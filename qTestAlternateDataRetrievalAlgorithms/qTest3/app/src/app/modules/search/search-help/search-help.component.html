Instructions

Using views

  • Use buttons in header bar to switch between Search, Platforms, Sources, Kits and Kit Comparison views.
  • Search filters can be set/modified/saved/loaded from any of the above views
  • Kit Comparison view is drillable to test case names via Sources on the Y-axis
  • Custom values for X- and Y- axis breakdowns can be configured from any pivot view using the options menu
  • Use 'Hide Filter Panel'/'Show Filter Panel' to toggle filter panel visibility

Instructions for search

Option 1: Search using filters

  1. Select search criteria*
  2. Click 'Search'
  3. Using Column Selector, select output columns (optional)
  4. Click 'Display Records' (optional)
  5. Note: column selector is only available on the Search view
  • * Right-hand side values are case-insensitive; key names are case-sensitive

Option 2: Search using saved search

  1. Select a saved named search from the "Load Search" select list
  2. Search will automatically load (results and search criteria) when selected

Option 3: Search using URL 'query' parameter

  1. Pass the name of a saved search using the 'query' parameter, e.g. http://trdb.iscinternal.com/app/#/?query=Olga%20Search%20HSPD%20Selenium
  2. Note: a search name containing spaces can be pasted directly into the url
  3. Search is immediately executed, and the search parameters are re-populated in the UI

Option 4: Search using search string

  1. After a search is performed, a string representing the search parameters is displayed under 'Search string'
  2. Copy this string to clipboard
  3. Paste into input field labeled "Paste saved search string:"
  4. Click 'String search' (optional)

To save current search:

PLEASE NOTE: new searches cannot be saved when connecting via one of the Mirror servers
  1. Execute a search using any of the above options
  2. Enter name of search in "Save search as" input box (will automatically be pre-loaded if search executed using a saved search)
  3. Modify search criteria and/or display columns (optional)
  4. Click 'Save search as'*
  • *If a saved search with the same name exists, it will be overwritten.
